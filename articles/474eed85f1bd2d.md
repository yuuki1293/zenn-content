皆さんこんにちは。

F#っていい機能がたくさんありますよね。いい機能のうちの一つに**カリー化**[^1]があります。カリー化によって関数の部分適用が簡単になったりといいことずくめですね、パイプライン演算子`|>`との相性は言うまでもないでしょう。

話は変わりますが、[haskell](https://www.haskell.org/)という言語において、すべての関数は1個の引数しか受け取りません。カリー化されていない関数は部分適用を簡単に行えないからですね。F#のベースとなった[OCaml](https://ocaml.org/)も1個の引数しか受け取れませんが、F#はどうでしょうか？

## 関数の定義
F#で4種類のadd関数を定義してみます。
```fsharp:add.fs
let add_1 (a: int) (b: int): int = // int -> int -> int
    a + b

let add_2 (a: int, b: int): int = // int * int -> int
    a + b

let add_3 ((a, b): int * int): int = // int * int -> int
    a + b
    
let add_4 ((a, b): struct (int * int)) = // struct (int * int) -> int
    a + b

add_1 1 2    |> printfn "%d"
add_2 (1, 2) |> printfn "%d"
add_3 (1, 2) |> printfn "%d"
add_4 (1, 2) |> printfn "%d"
```
```shell:実行結果
3
3
3
3
```
コメントはエディター[^2]で表示された型です。  
`add_1`は`int`型を受け取り`int -> int`型の関数を返す関数で、1個の引数を受け取る関数になっていますね。  
`add_2`、`add_3`は`int * int -> int`型で、`int * int`型を受け取り`int`型を返す関数で、1個の引数を受け取るように見えます。見えますと言うのは、`add_2`は実際には違うからです。  
`add_4`は`struct (int * int)`型を受け取り`int`型を返す関数で、これも1個の引数を受け取る関数ですね。

  
`add_2`と`add_3`が違う型であることは、メソッドのオーバーロードの条件より確認できます。
```fsharp:AddClass.fs
type AddClass =
    static member add (a: int, b: int): int = // int * int -> int
        a + b

    static member add ((a, b): int * int): int = // int * int -> int
        a + b
```
同名のメソッドは同じ型で定義できませんが、`AddClass`はコンパイルエラーとはなりません。もちろんこんなわかりずらい方法でオーバーロードする利点はないです。  
それぞれのメソッドは以下のように呼び出せます。  
```fsharp:call_add.fs
AddClass.add (1, 2) // 上が呼ばれる

AddClass.add ((1, 2)) // 下が呼ばれる

let param: int * int = 1, 2
AddClass.add param // 下が呼ばれる
```
明示的にタプルを渡すと下のメソッドが呼ばれるようです。  
オーバーロードされていない場合はどの方法でも関数を呼び出せます。  

## ILで確認してみる
多分これが一番早いと思います。  
.NET 言語で書かれたプログラムは、一度中間言語（IL =  Intermediate Language）にコンパイルされます。  
`AddClass`がどのようにコンパイルされるか見てみましょう。  
@[gist](https://gist.github.com/yuuki1293/0156dd060fb26f2eab70fbe8c31d079f#file-addclass-il-L20)
```il

.method public static int32

    ) cil managed
{
～略～
} // end of method AddClass::'add'
～略～
```
https://gist.github.com/yuuki1293/0156dd060fb26f2eab70fbe8c31d079f
上の`add`メソッドは`int32`型の引数`a`と`b`を持っていて、下の`add`メソッドは``System.Tuple`2<int32, int32>``型の引数`ab`を持っていることが確認できます。  

## 結論
`func (a, b) = ...`の形式で定義される関数・メソッドは実際には2つの引数を受け取ります。  

## なぜそうなる？
もし
皆さんご存知の通り、F#はC#と相互に参照できます。  
```fsharp:TupleMethod.fs
namespace Fs

type TupleMethod =
    static member method (arg: int * int) = // System.Tuple<int, int>
        let sum = fst arg + snd arg
        printfn $"%d{sum}"
```
```csharp:Main.cs
using Fs;

TupleMethod.method((1, 2).ToTuple());
```
```shell:実行結果
3
```
C#上で`(1, 2)`は`ValueTuple<int, int>`型となるため、`ToTuple()`メソッドで`Tuple<int, int>`型に変換する必要があります。
毎回こんな書き方で呼び出すのは大変ですよね。
